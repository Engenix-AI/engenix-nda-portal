ENGENIX CLOUDFLARE → GOOGLE WORKSPACE RELAY V16

WHY THIS VERSION
The browser now posts only to ENGENIX at /api/demo.
Cloudflare forwards the request to Google Apps Script.
This removes the iPhone Safari download behavior and keeps the Google URL out of the browser.

FILES TO DEPLOY
- index.html
- demo.html
- script.js
- styles.css
- images/assets
- functions/api/demo.js

IMPORTANT CLOUDFLARE STRUCTURE
The functions folder must be at the Cloudflare Pages PROJECT ROOT.
It cannot sit only inside the public /beta/ folder.

GOOGLE APPS SCRIPT UPDATE
1. Open your ENGENIX Apps Script project.
2. Replace Code.gs with google-apps-script/Code.gs from this package.
3. Save.
4. Deploy → Manage deployments.
5. Tap the pencil/Edit icon.
6. Version → New version.
7. Deploy.

The existing /exec URL remains the same.

WEBSITE DEPLOYMENT
Upload this package and redeploy the Cloudflare Pages project.

TEST
1. Open https://www.engenix.co/beta/
2. Submit the homepage Private Access form.
3. Confirm:
   - No Safari download prompt
   - Website shows Access Requested
   - New Google Sheet row appears
   - Internal Workspace email arrives
   - Prospect confirmation arrives

ENDPOINT
Browser: /api/demo
Cloudflare relay: functions/api/demo.js
Google Workspace: existing Apps Script /exec URL
